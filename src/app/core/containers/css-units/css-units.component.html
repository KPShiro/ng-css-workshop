About CSS units

Absolute length units

The following are all absolute length units â€” they are not relative to anything else, and are generally considered to always be the same size.

AWESOME NEWS: pixels (px) are relative to the viewing device. For low-dpi devices, 1px is one device pixel (dot) of the display. For printers and high resolution screens 1px implies multiple device pixels.
Unit Note
cm centimeters
mm millimeters
in inches
px pixels (1px = 1/96th of 1in)
pt points (1pt = 1/72 of 1in)
pc picas (1pc = 12 pt)

Relative length units

Relative length units are relative to something else, perhaps the size of the parent element's font, or the size of the viewport. The benefit of using relative units is that with some careful planning you can make it so the size of text or other elements scales relative to everything else on the page.

Unit Relative to
em
  • - Font size of the parent (in the case of typographical properties like font-size)
  • - Font size of the element itself (in the case of other properties like width)
rem Font size of the root element.
vw 1% of the viewport's width.
vh 1% of the viewport's height.
vmin 1% of the viewport's smaller dimension.
vmax 1% of the viewport's larger dimension.

Usefull information

Which one should be used?

Well... it depends what you want to achieve. If you want to have an element that adapts to, for example, browser settings - use rem or em. If you are not worried about scaling and want to be sure that certain thing stays always the same - use px.

What is recommended?

I recommend to use rem in most cases. This way, accessability of the page/app will not suffer if users will decide to change their browser or device settings.

Things to remember...

  • Always respect user's accessability settings
  • Try to use rem when it's possible - almost every UI element should adjust to the user settings
  • If you are working with fixed-size elements like borders or shadows, px may be the better choice
  • Try not to hardcode width or height of the element - they should be "flexible" by default, quite often it will require less changes in the future
