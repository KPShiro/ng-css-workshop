Absolute length units

The following are all absolute length units ‚Äî they are not relative to anything else, and are generally considered to always be the same size. Most of these units are more useful when used for print, rather than screen output. For example, we don't typically use cm (centimeters) on screen.

Unit Note
cm centimeters
mm millimeters
Q Quarter-millimiters
in inches
pc picas (1pc = 12 pt)
pt points (1pt = 1/72 of 1 in)
px pixels (1px = 1/96th of 1 in)

Relative length units

Relative length units are relative to something else, perhaps the size of the parent element's font, or the size of the viewport. The benefit of using relative units is that with some careful planning you can make it so the size of text or other elements scales relative to everything else on the page.

Unit Note (relative to)
em
  • - Font size of the parent (in the case of typographical properties like font-size)
  • - Font size of the element itself (in the case of other properties like width)
ex x-height of the element's font.
ch The advance measure (width) of the glyph "0" of the element's font.
rem Font size of the root element.
lh Line height of the element.
rlh Line height of the root element. When used on the font-size or line-height properties of the root element, it refers to the properties' initial value.
vw 1% of the viewport's width.
vh 1% of the viewport's height.
vmin 1% of the viewport's smaller dimension.
vmax 1% of the viewport's larger dimension.
vb 1% of the size of the initial containing block in the direction of the root element's block axis.
vi 1% of the size of the initial containing block in the direction of the root element's inline axis.
svw, svh 1% of the small viewport's width and height, respectively.
lvw, lvh 1% of the large viewport's width and height, respectively.
dvw, dvh 1% of the dynamic viewport's width and height, respectively.

The Pixel Unit

What is the pixel unit? Most probably, during the research, you will find very different definitions. Some of the sources will define it as a "measurement that never changes", some as "non linear-angular measurement" and others as "pixel density of the device".

96px

The square above should be 1 inch x 1 inch (96px x 96px). If you compare the size of it on 2 different devices (e.g. your computer and smartphone), you will easily see that there is a difference. Also, measured with a ruler it will not match as well. But why is that? Isn't the pixel unit absolute?

In short, it is all related to the anchor unit. For print, CSS will apply the same rules known to you, so 1cm on the printed page is exactly 1cm. The situation changes for electronic devices (like tv, notebook, and smartphone...). Because of a different expected viewing distance, there is a difference in the reference pixel.

em & rem units

Both em and rem units are categorized as being relative units that are relative to the font size. In the case of the em unit, it is relative to the font size of the element it is assigned to, unless it is assigned to the font-size property, in which case it will be relative to the font size of the parent.

rem instead of being relative to the font size of the parent, it is being relative to the font size of the root element. Root element is the html element, which has by default font size of 16px (or rather 100% font size of native browser settings).
